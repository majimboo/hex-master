Schema Guide

Describe binary structures with a compact DSL.

Hex Master schemas define endianness, named structs, arrays, and offset-based fields. Apply a schema at a base offset to parse binary data into a structured tree.

Syntax

Core schema shape

Every schema has an endianness, zero or more named structs, and one root struct.

endian little

struct Entry {
  u32 id
  u16 flags
  u16 value
}

root Header {
  bytes[4] magic
  u16 version
  u16 entry_count
  Entry[entry_count] entries
}

Examples

Arrays and offset-based fields

endian little

struct Entry {
  u32 id
  u16 flags
  u16 value
}

root Header {
  bytes[4] magic
  u16 version
  u16 entry_count
  u32 entries_offset
  Entry[entry_count] entries @entries_offset
}

In this example, entries is an array of Entry records. The array length comes from the earlier entry_count field, and the array starts at the offset stored in entries_offset.

Rules

Supported constructs

Scalars

  • u8, u16, u32, u64
  • i8, i16, i32, i64
  • f32, f64

Containers

  • Named struct blocks
  • One root block
  • Byte arrays with bytes[n]
  • Struct arrays with Type[count]

References

  • Array counts can use an earlier scalar field
  • Fields can start at an earlier scalar offset using @field_name
  • Base offset is supplied when running the schema

The current DSL is intentionally small. It does not yet support dotted paths, unions, conditionals, or complex expressions. Keep dependent fields defined after the scalar values they depend on.
